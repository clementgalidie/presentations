Naming convention

  • Use CamelCase for classes and modules.
  • Use snake_case for symbols, methods and variables
  • The names of predicate methods (methods that return a boolean value) should end in a question mark
  • The names of potentially dangerous methods (methods that modify self or the arguments, exit! etc.) should end with an exclamation mark.

Les Test unitaires

  • require 'test/unit'

    La librairie de test unitaire est dans la librairie standard. Il suffit de la require en début de fichier.

  • Herite de Test::Units::TestCase

    Toutes classes de Test unitaire doit hériter de Test::Units::TestCase. La classe doit aussi finir par ...Test

  • Nom de methode commencant par test_

    Toutes les methodes commencant par test_ seront lancé pour faire des tests. Les test ne commancant pas par ce mot clé en seront pas executé par les tests.

  • assert_equal

    L'assertion de base qui vérifie que les 2 arguments sont égaux l'un par rapport à l'autre

  • assert

    Permet de vérifier que l'argument donnée est true

  • setup

    La méthode setup est executé avnat chaque test. Elle permet de faire des initialisation de données

  • teardown

    Cette méthode si elle est defini est executé après chaque test. Elle sert a supprimer des données créés durant les tests.

Test unitaire

require 'car' require 'test/unit' class CarTest < Test::Unit::TestCase def setup @car = Car.new end def test_init assert !@car.running? end end
